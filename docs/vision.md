# CloudMind 愿景与定位（North Star）

> 状态：定盘 · 2026-06-07
> 这是 CloudMind 的**设计宪法**。方向类决策以本文为准；架构（[`memory-layer-architecture.md`](memory-layer-architecture.md)）与路线图（[`memory-layer-roadmap.md`](memory-layer-roadmap.md)）服从本文。

---

## 一句话定位

CloudMind 是一个**开源、数据主权、个人【永久】记忆层**——「**AI 时代·人的记忆层**」。

记忆的**主体是「人」**（你的经历、偏好、知识、做事方式），**AI 只是随时可读它的消费者**，不是主人。AI 让「记录」的门槛降到极低，于是你的 everything 都能自部署地存下来，并被任何 AI 读取。

**不是**「AI 的记忆层」（mem0 / Letta 那类：主体是 agent、为上下文预算而压缩遗忘、卖开发者 API）。我们从不与之竞争；能力可借鉴，定位完全不同。

## 北极星

> 一个人的**永久**记忆——假设 50 年后底层云仍在，CloudMind 仍能读出今天这个决定。
> 它承载你愿意记下的一切（这些只是**例子，不是要做的功能**）；真正要做的是**让任何「检索词/线索」进来时，把内容检索好**。

**解决的核心痛点**：每次跟 AI 对话都要重复一遍「我是谁、我的背景、我的要求」。有了它，AI 先读你的记忆，再懂你地回答。论点：**更懂你的 AI 才是终局，而不是每次都失忆的 AI。**

## 成功的定义

开源·主权工具，被同类 prosumer / 开发者采用，且**比厂商锁定的记忆（ChatGPT / Claude Memory，故意不自部署、不可移植）更好用**。**不追规模营收、不做 SaaS。**

## Founding 原则（由「永久 + 主权 + 以人为主体」反推）

1. **保存 > 抽取；L1 原始记录神圣不可变（50 年时间胶囊）。**
   L2（知识图谱）/ L3（记忆面）是**派生的、可丢弃、可由未来更强的 AI 重算**的层。当下重点是**忠实保存原始**，而非把抽取做到完美——未来 AI 会抽得更好。「实体关系」本就是人为抽象，故属可重算的 L2，不是地基。

2. **永久性 > 平台。**
   必须能在 Cloudflare 倒闭后存活 ⇒ **开放、自描述、可完整导出**是目的本身，不是迁移对冲。主权 = 你的记忆比任何厂商活得久。（导出可推迟实现，但在任何「正式灌库」里程碑前必须就位。）

3. **不自动遗忘。**
   永久记忆层按定义**不按年龄/低频自动忘**。recency 仅作**排序信号** + 可选「工作集」视图；删除**只**走用户显式 `forget` 动词（且默认软删/可回溯）。一致性修复（失效漂移边、去重）只软标记（`expired_at`/`superseded_by`），不毁数据。

4. **主权也用在 AI 侧：MCP-first。**
   记忆**不锁定单一 AI**——任何 AI 客户端经 MCP 都能读。「更懂你的 AI」靠**可移植的你**实现，而不是赌某家 AI 永远最强。

5. **以人为中心 + 权限门控。**
   系统的中心是「你这个人」。私密信息（收入、健康…）由可见性字段（`aiVisibility`）门控，控制不同 AI 客户端能否读取。

## 检索观：我们还是 RAG 吗？是——但破的是 *naive* RAG

「以检索为核心」不是退而求其次，**它本来就是记忆的样子**（人脑回忆也是线索触发的检索）。要破的从来不是「检索」，而是**幼稚的检索**（切块→嵌入→cosine 取 top-K→塞进去）。

差异不在用不用检索，而在检索的**前后**：
- **写入侧智能**：调和矛盾（bi-temporal 失效）、去重、蒸馏成干净事实 —— 让你搜「我住哪」只返回当前真相，而不是一堆矛盾旧块。
- **检索质量工程**（已落地，P1）：结构/token 感知切块 · `dense 语义 + lexical BM25/FTS5`（含中文 trigram）混合 · 跨通道融合 · `bge-reranker` 交叉编码重排 · MMR 去近重复 · Vectorize 原生 metadata 过滤 · eval 棘轮防回归。
- **永久 + 出处 + 个人化与权限**。

**「就是个 RAG 脚本」 vs 「记忆层」的差距，全在这些「前后」，不在「是否检索」。**

### 检索 = 模拟人脑的多种回忆模态（产品的真功能轴）

> 关键：按**检索模态**做能力（通用、跨任何内容），**不**按「日记/习惯/荣誉/情绪」这类具体场景做（无通用性）。内容只是喂进来的数据。

| 模态 | 大白话 | 机制 | 现状 |
|---|---|---|---|
| **M1 语义/关键词** | 按意思/关键词找 | 混合 RAG + 重排（P1） | ✅ 已有 |
| **M2 个人/常驻** | 「关于我、跟这事相关的」（先读你的背景再答） | 带主题标签 + 可见性的个人条目，语义检索整捆返回；**不依赖实体** | ← 当前最高优先（Phase A） |
| **M3 时间** | 按日期 / 相对事件锚定 | L1 情节 + 时间范围检索 | ❌ 待补 |
| **M4 联想** | 线索勾出关联（关系） | L2 图遍历 | ⚠️ 图检索已部分集成 |
| **M5 评价/超级值** | 「最…的那个」 | 入库打评价维度标签 + 超级值排序；novel 维度用 map-reduce 逐条打分 | ❌ 待补 |
| **M6 聚合/算一遍** | 「几件大事 / 共几次 / 趋势」 | 对一批**计算**（摘要/计数），非点查 | ❌ 待补（真·非 RAG 超能力） |

**核心二分**：**点查（RAG）** vs **对一批算一遍（M6 超级值/计数/摘要）**——后者 RAG 天生做不到，最体现护城河。

「AI 读懂你的背景」（M2）的主力是**带主题标签、可语义检索、带可见性的个人条目**，**不是图谱**；图谱只服务 M4 这一小撮关系类查询。

## 由愿景反推的优先级（详见 roadmap）

- **先做**：M2 个人化 recall（remember/recall + 可见性）—— 把我们和「一个 RAG 脚本」彻底拉开。
- **再做**：补全其余检索模态 M3/M5/M6（M4 已部分）。
- **后做**：兑现主权（导出）、`forget`、图谱整合/社区（按需）。
- **砍**：自动按年龄遗忘（违原则 3）。
- **降级**：L2 图谱 consolidation/community（锦上添花，非体验驱动）。
